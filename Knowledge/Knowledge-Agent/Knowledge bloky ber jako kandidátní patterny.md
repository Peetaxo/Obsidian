# Knowledge bloky ber jako kandidátní patterny

## Trigger
- výběr relevantních patternů
- review knowledge báze
- prevence knowledge debt
- aplikace YouTube poznatku v projektu

## Core rule
- Pokud pattern není ověřený v projektu → ber ho jako kandidáta, ne jako povinné pravidlo

## Decision rules
- Pokud pattern není ověřený v praxi → posuď kontext před použitím
- Pokud se pattern osvědčí → zvyš jeho důvěru v human vrstvě
- Pokud se pattern opakovaně nehodí → zúž ho, slouč nebo archivuj
- Kandidátní patterny rozšiřují možnosti, ale nesmí řídit práci naslepo
- Ověření v praxi zvyšuje důvěru víc než popularita videa
- Neužitečné bloky zvyšují knowledge debt

## Use for
- vývoj webové aplikace
- výběr agentického workflow
- lint knowledge báze
- rozhodnutí o novém bloku

## Avoid
- YouTube tip jako ověřená best practice
- aplikovat pattern bez kontextu
- držet slabý blok jen kvůli zdroji
