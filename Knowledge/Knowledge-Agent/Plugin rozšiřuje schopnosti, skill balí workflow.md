# Plugin rozšiřuje schopnosti, skill balí workflow

## Trigger
- Volíš mezi pluginem a skillem
- integrace externí služby
- reusable workflow

## Core rule
- Plugin/MCP/API přidává schopnost; skill definuje postup použití

## Decision rules
- Pokud agent nemá přístup ke službě → potřebuje plugin, MCP nebo API integraci
- Pokud agent přístup má, ale pokaždé neví postup → vytvoř skill
- Pokud workflow kombinuje více nástrojů → skill popíše pořadí a výstup
- Plugin řeší schopnost
- Skill řeší postup
- Dobrý skill obalí plugin, MCP nebo API do stabilního workflow

## Use for
- napojení kalendáře, emailu, Figmy nebo Typefully
- tvorbě YouTube researcher workflow
- balení MCP do použitelného agentického postupu
- vysvětlování rozdílu mezi tools, skills a plugins

## Avoid
- Instalovat další plugin, když problém je jen chybějící postup, nebo psát skill, když agent vůbec nemá potřebný přístup
